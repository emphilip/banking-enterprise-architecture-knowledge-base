# Regulatory Taxonomy Manager

**Definition.** Regulatory Taxonomy Manager is the technology sub-capability that manages regulator taxonomies, templates and report definitions (e.g. XBRL/DPM) and keeps them current with changing rules across regimes.
**Also known as:** Taxonomy Manager, Reporting Template Manager, XBRL Taxonomy Service, Regulatory Rules Library.

## Relationships
- Regulatory Taxonomy Manager is defined in the Data & Analytics domain.
- Regulatory Taxonomy Manager is derived from Regulatory Reporting Engine.
- Regulatory Taxonomy Manager depends on Regulatory Data Model.
- Regulatory Taxonomy Manager is related to Regulatory Reporting.

## Details
Regulatory Taxonomy Manager holds the machine-readable definitions of regulatory returns — EBA / EIOPA XBRL taxonomies and their underlying Data Point Model (DPM), plus national template specifications. It maps report cells to data points and validation rules, and ingests taxonomy releases so templates stay aligned with each reporting framework version as regulators publish changes. By separating regulator-published taxonomy from the bank's calculation logic and data model, it lets new or amended returns be onboarded as configuration rather than code.
