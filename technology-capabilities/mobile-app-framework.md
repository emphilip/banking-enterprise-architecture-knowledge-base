# Mobile App Framework

**Definition.** Mobile App Framework is the technology sub-capability providing the native and hybrid mobile app runtime, SDKs and design system that deliver banking journeys on iOS and Android, including device binding and app shielding.
**Also known as:** Mobile SDK Platform, Mobile Banking Framework, Multi-Experience Framework.

## Relationships
- Mobile App Framework is defined in the Channels & Engagement domain.
- Mobile App Framework is derived from Digital Channel Platform.
- Mobile App Framework depends on API Management.
- Mobile App Framework is related to Customer Identity.

## Details
Mobile App Framework packages native iOS (Swift) and Android (Kotlin) runtimes alongside cross-platform options, exposing reusable widgets, a shared design system and secure on-device storage. It applies device binding, certificate pinning and runtime application self-protection (RASP) app shielding to harden the client, and consumes back-end services exclusively through managed APIs rather than direct core access. Vendor stacks such as Backbase Engagement Banking and Temenos Infinity ship modular SDKs so banks can compose journeys without rebuilding the shell.
