# Native Mobile SDK

**Definition.** Native Mobile SDK is the technology sub-capability providing an embeddable native iOS and Android software development kit that exposes banking widgets, secure storage and biometric APIs for in-house and partner apps.
**Also known as:** Mobile Banking SDK, Embeddable Mobile SDK.

## Relationships
- Native Mobile SDK is defined in the Channels & Engagement domain.
- Native Mobile SDK is derived from Mobile App Framework.

## Details
Native Mobile SDK ships compiled iOS (Swift) and Android (Kotlin) libraries that drop pre-built, themeable banking screens and components into a host app, with APIs for Keychain/Keystore-backed secure storage, biometric (Face ID / fingerprint) unlock and device binding. It lets a bank embed regulated journeys inside its own super-app or a partner's app without re-implementing security plumbing. Backbase and Alkami publish such SDKs so engineering teams reuse certified components rather than building native banking UIs from scratch.
