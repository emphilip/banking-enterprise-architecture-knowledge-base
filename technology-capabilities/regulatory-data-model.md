# Regulatory Data Model

**Definition.** Regulatory Data Model is the standardised, harmonised data-model technology sub-capability that ingests and maps source data for consistent, granular regulatory reporting across jurisdictions.
**Also known as:** Harmonised Data Model, Granular Data Layer, Reporting Data Model, Reg Data Layer.

## Relationships
- Regulatory Data Model is defined in the Data & Analytics domain.
- Regulatory Data Model is derived from Regulatory Reporting Engine.
- Regulatory Data Model depends on Data Governance.
- Regulatory Data Model is related to Data Warehousing.

## Details
Regulatory Data Model provides a single, granular input layer — the Regnology Abacus data model is the archetype — into which source ledger, risk and reference data are mapped once and reused across many returns. This template-free, granular-data approach lets regulators and banks derive reports from atomic records rather than pre-aggregated templates, supporting initiatives like BIRD/IReF. Consistent definitions, supported by enterprise data governance, prevent the same fact being computed differently for different reports and make figures reconcilable to source.
