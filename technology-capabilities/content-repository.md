# Content Repository

**Definition.** Content Repository is the secure, metadata-driven central store for documents and content with version control and permission enforcement across the content lifecycle.
**Also known as:** Content Store, Document Repository, ECM Repository.

## Relationships
- Content Repository is defined in the Data & Analytics domain.
- Content Repository is derived from Document Management.
- Content Repository is related to Archival & Retention Service.
- Content Repository is related to Records Management Service.

## Details
Content Repository is the foundation of an enterprise content management (ECM) system: a managed store that holds documents and unstructured content together with rich metadata that describes and classifies each item. Rather than relying on folder location, retrieval is metadata-driven, so the same document can be found by customer, account, product, or document type. The repository enforces access permissions, maintains version history with check-in and check-out, and provides an audit trail of who viewed or changed content. It is the shared store that capture, workflow, records, and retention capabilities all operate against in a banking ECM platform.
