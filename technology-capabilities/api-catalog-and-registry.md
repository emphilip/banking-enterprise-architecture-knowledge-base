# API Catalog & Registry

**Definition.** API Catalog & Registry maintains a centralized, searchable inventory of APIs, events, and their specifications, schemas, and metadata for discovery, reuse, and governance.
**Also known as:** API Registry, API Catalogue, API Marketplace Index.

## Relationships
- API Catalog & Registry is defined in the Integration & APIs domain.
- API Catalog & Registry is derived from API Management.
- API Catalog & Registry is related to Developer Portal.

## Details
API Catalog & Registry stores each API's OpenAPI/AsyncAPI definition, version history, owning team, lifecycle status, classification, and dependency links so engineers can find and reuse existing services instead of duplicating them. It often pairs with a schema registry that validates event and message contracts for compatibility. The catalog is the source of truth the developer portal renders for discovery, and it underpins governance reporting on coverage, ownership, and deprecation.
