# Regulatory Submission Gateway

**Definition.** Regulatory Submission Gateway is the connectivity technology sub-capability that submits returns to regulators and ingests feedback and acknowledgements via APIs and regulator portals.
**Also known as:** Submission Gateway, Regulator Connectivity, Filing Gateway, Reporting API Gateway.

## Relationships
- Regulatory Submission Gateway is defined in the Data & Analytics domain.
- Regulatory Submission Gateway is derived from Regulatory Reporting Engine.
- Regulatory Submission Gateway depends on Report Validation Service.
- Regulatory Submission Gateway is related to Regulatory Reporting.

## Details
Regulatory Submission Gateway handles secure last-mile delivery of validated returns to each regulator's channel — portal upload, secure file transfer, or increasingly API-based submission as supervisors move to API-driven, granular reporting. It packages returns in required formats (XBRL instances, fixed-format files), signs and transmits them, and ingests acknowledgements, rejections and validation feedback to update filing status. Tracking of deadlines, resubmissions and receipts gives a complete record of what was filed, when and with what regulator response.
