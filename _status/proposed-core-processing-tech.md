## Skips (candidate L3 already exists in registry, or parent already has the L3)

- Real-Time Posting Service — already exists as L3 under Transaction Posting Engine.
- Least-Cost Routing Service — already exists as L3 under Payment Routing Engine.
- SWIFT Connectivity Adapter — already exists as L3 under Scheme Connectivity Gateway.
- ISO 20022 Translation Service — already exists as L3 under Payment Message Transformer.
- EMV Cryptogram Validation, Stand-In Processing Service, 3-D Secure Service — already exist as L3 under Card Authorization Engine.
- Tiered Interest Calculator — already exists as L3 under Interest & Charges Engine (so no second L3 added there beyond Accrual Calculation Service).
- Tax-Loss Harvesting Service — already exists as L3 under Rebalancing Engine.
- Pre-Trade Compliance Engine — already exists as L3 under Order Routing Engine.
- Suitability Assessment Service — already exists as L3 under Risk Profiling Engine.
- Income Verification Service — already exists as L3 under Affordability Assessment Engine.
- Strategy Design Studio / Reject Inference Model — already exist as L3 under Decision Rules Engine / Credit Scoring Service.
- Multi-Currency Ledger Service, Intercompany Accounting Service — already exist as L3 under Journal Processing Engine / Subledger Accounting.
- In-House Banking Service, Financial Data Quality Service — already L3 (under Treasury Management System / Financial Consolidation Platform).
- Avoided alias clashes with existing L2 Accounting Rules Engine, which carries the aliases "Posting Rules Engine" and "Document Splitting Service". The proposed L3 under Transaction Posting Engine was therefore named "Posting Determination Service" (not "Posting Rules Engine"), and the proposed L3 under Accounting Rules Engine was named "Document Splitting Engine" (not "Document Splitting Service"), to keep both globally unique.
