# Routing (INT-RTG)

**L0:** [[INT Interaction Mgmt.|Interaction Mgmt.]]

Routing determines which resource — agent, skill group, or automation — should handle each incoming interaction and steers it there.

## L2 Capabilities

| ID | Capability | Summary |
|---|---|---|
| INT-RTG-01 | [[INT-RTG-01 Intelligent\|Intelligent]] | Data-driven routing that selects the best destination using customer, context, and intent signals |
| INT-RTG-02 | [[INT-RTG-02 Skills-Based\|Skills-Based]] | Matches interactions to agents based on skill, language, and proficiency profiles |
| INT-RTG-03 | [[INT-RTG-03 Predictive\|Predictive]] | Uses predictive models to anticipate need and route proactively or pre-emptively |
| INT-RTG-04 | [[INT-RTG-04 Priority\|Priority]] | Elevates and sequences interactions by urgency, value, or SLA before assignment |
