# Conduct Root Cause

**Definition.** Conduct Root Cause analyses drivers across complaints, identifies systemic root causes and raises improvement actions and MI reporting.
**Also known as:** Root Cause Analysis Step, Analyse Drivers.

## Relationships
- Conduct Root Cause is defined in the Complaint Handling process.
- Conduct Root Cause causes Close Complaint.
- Conduct Root Cause depends on the Analytics Platform capability.
- Conduct Root Cause mentions Quality Assurance Reviewer.
- Conduct Root Cause mentions Complaint Record.

## Details
The Quality Assurance Reviewer analyses drivers across complaints. Inputs are the Final Response Letter; outputs are a root cause report. Controls include root-cause analysis and complaints MI/regulatory reporting.
