# Legacy Mainframe Core

**Definition.** Legacy Mainframe Core refers to the category of in-house or older core banking systems that run on mainframe hardware, typically written in COBOL and processing accounts and transactions in batch.
**Also known as:** COBOL Core, Mainframe Core.

## Relationships
- Core Banking Processing depends on Legacy Mainframe Core.
- Legacy Mainframe Core mentions 10x Banking.
- Legacy Mainframe Core is related to FIS Profile.

## Details
Legacy Mainframe Core systems are commonly built on IBM Z mainframes with COBOL programs and CICS transaction processing, often accumulated over decades. They are characterized by batch-oriented end-of-day processing, deep customization, and scarce specialist skills. These traits, along with high operating costs and limited real-time capability, make them frequent targets for core modernization to cloud-native platforms.
